# code-map v0.2.0 ‚Äî CLI-Only Agent Toolkit

**Date:** 2026-05-06
**Goal:** Make `code-map` a complete CLI surface for AI agents that shell out (no MCP, no daemon, no LSP). Every primitive an agent needs is one shell command.

**Research basis:** `EdwardAstill/warden:.warden/research/2026-05-06-code-map-feature-validation/REPORT.md`. Validated against 9 production agents (Aider, Cline, Continue, Cursor, Plandex, Cody, Roo Code, OpenCode, Goose) and 14 ablation studies (AutoCodeRover, SWE-agent, RepoGraph, LocAgent, CatCoder, RepoBench, RepoQA, SWE-Pruner, etc.).

**Scope locked:** CLI-only. No MCP server. No daemon. No embeddings.

## What v0.1.0 already does (keep)

- Tier-1 langs: Python, TS/JS, Rust via tree-sitter.
- Symbols: `name`, `qualified_name`, `file_path`, `line_start/end`, `kind`, `signature`.
- Edges: `import`, `calls`, `inherits`, `implements`, `references` with `certainty` + `source_type`.
- Subcommands: `build`, `refresh`, `query`, `render`.
- Storage: SQLite (graph.db) + JSONL mirrors + manifest.json.
- Token-budgeted MAP.md (PageRank-ordered) + per-package markdown.
- Honesty contract on edges + in-artifact README.

## v0.2.0 ‚Äî what's new

### New subcommands

- `code-map source <symbol>` ‚Äî fetch source code for a symbol (the most-issued agent query).
- `code-map find <pattern>` ‚Äî fuzzy/substring symbol search by name or qualified_name.
- `code-map status` ‚Äî `{built_at, files_total, files_changed_since_build, stale}`. Lets agents decide whether to refresh.
- `code-map outline <file>` ‚Äî tree-style outline with class > method nesting.
- `code-map context <file>:<line>` ‚Äî reverse lookup: which symbol contains this line?
- `code-map schema` ‚Äî emits JSON Schema 2020-12 for the artifact contract.

### New flags on existing subcommands

- `--format json|markdown` on every query (default JSON; markdown is for direct prompt-paste).
- `--limit N` on every query (default 50; hard-stop 100 unless `--no-limit`).
- Symbol globbing: `b::*`, `*greet*` translate to SQL LIKE.

### Output enrichment

- All query results carry a `score: float` column (PageRank √ó inverse-depth for blast-radius; PageRank for callers/callees).
- Query results sorted by `score` desc, then qualified_name asc.

### CLI hygiene

- Distinct exit codes: `0` ok, `2` no artifact (run `build` first), `3` symbol/file not found, `4` stale.
- Stderr for diagnostics; stdout strictly machine-parseable (JSON or markdown).

### Docs

- Expanded README: concrete examples, Claude Code integration recipe, comparison vs aider repo-map / SCIP / ctags.
- CHANGELOG.md.
- GitHub Actions CI (ruff + pytest on push).
- Tag v0.2.0.

## What v0.2.0 explicitly does NOT add

(Failed the "agents demonstrably consume" test in research; do not pollute v0.x.)

- MCP server, daemon, LSP.
- Embeddings / vector search.
- Per-symbol doc-strings.
- Parsed type AST (textual signatures stay).
- Visibility flags (pub/private/export).
- def-vs-decl-vs-impl distinction.
- Test‚Üîcode links.
- Side-effect / mutation annotations.
- Cross-language edges.
- Macro expansion.
